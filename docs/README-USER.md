## OBS Overlay Kill（一般ユーザー向け）

開発環境は不要です。**配布された `OBS-Overlay-Kill.exe` を実行して OBS に取り込む**だけで使えます。

> このリポジトリにある `.bat`（バッチファイル）は主に**開発者向け**です。配布 exe を使うだけなら基本的に不要です。

### 使い方（最短）

- **起動**: 配布物を展開 → `OBS-Overlay-Kill.exe` を起動
- **表示**: ブラウザで `http://localhost:4173/overlay` を開く（既定ポート）
- **OBSに取り込み**: OBS で「ブラウザソース」または「ウィンドウキャプチャー」で上記ページを取り込む
- **設定**: `/overlay` 内の設定パネルで HP / 攻撃 / 回復 / 効果音 などを調整
- **保存**: 設定パネルの「保存」を押す（自動で保存されます）
- **終了**: 起動した黒いウィンドウで `Ctrl + C`

### 保存先（触ってよい範囲）

初回起動時、exe 内の設定・素材がここへコピーされます。

- `%LOCALAPPDATA%\OBS-Overlay-Kill\data\`
  - 効果音: `data\src\sounds\`
  - 画像・WebM: `data\src\images\`
  - 設定: `data\config\overlay-config.json`
  - カスタム技名（任意）: `data\config\customTechniqueNames.ts`

### 設定を初期化したい

- `data\config\overlay-config.json` を削除して exe を再起動すると初期設定に戻ります。

### ポート変更

PowerShell 例:

```powershell
$env:OVERLAY_PORT=8080
.\OBS-Overlay-Kill.exe
```

### PRO 機能の解放（月額サブスクリプション）

一部の設定（確率・抽選、自動返信、対人 PvP、表示位置の微調整など）は **PRO 機能** です。  
有料で使うには **月額契約（Stripe）** か、管理者からもらった **招待コード** のどちらかが必要です。

#### 課金タブを開く

1. `OBS-Overlay-Kill.exe` を起動したまま、ブラウザで `http://localhost:4173/overlay` を開く  
2. 画面の **テストパネル**（操作・設定・**課金** の切り替え）で **「課金」** を選ぶ  
3. 「機能の解放」パネルが表示されます  

ロックされた PRO 設定には「課金タブからログインし…」と表示されます。同じ **課金** パネルへ進んでください。

#### 全体の流れ

```
① ログイン / アカウント登録（メール → 届いたリンクを開く）
② 月額契約（Stripe）または 招待コード のどちらか
③ 「利用中」になれば PRO 設定が使える
```

**① は Stripe でも招待コードでも必ず先に行います。** 管理者から招待コードをもらった場合も、同じ手順でログインしてからコードを入力してください。

---

#### ① ログイン（アカウントをお持ちの方）

1. **課金** パネルの **「① ログイン / アカウント登録」** で **「ログイン」** を選ぶ  
2. **登録済みのメールアドレス** を入力 → **「ログインリンクを送信」**  
3. メールに届いた **リンクを、この PC の同じブラウザで** 開く（exe 起動中であること）  

**重要**

- リンクは **スマホで開かない**（`localhost` のため PC 上のブラウザだけ有効）  
- リンクを送った **のと同じブラウザ** で開く（Chrome で送ったなら Chrome で開く）  
- `email rate limit exceeded` と出たら **30〜60 分待つ**（連打しない）  
- 招待用メールだけ管理者に登録されている場合は、一度 **「新規登録」** が必要なことがあります  

---

#### ① 新規登録（初めての方）

1. 同じ **「① ログイン / アカウント登録」** で **「新規登録」** を選ぶ  
2. **メールアドレス** と **ユーザー名** を入力 → **「登録リンクを送信」**  
3. メールの **リンク** を開いて登録完了  

- ログイン後、ユーザー名が未保存と表示されたら **「ユーザー名を保存」** を押してください  
- **②（契約・招待コード）は、ユーザー名の保存が終わってから** 使えます  

---

#### ② 月額サブスクリプション（Stripe）

**②-A 月額サブスクリプション（Stripe）** から契約します。

| 契約の種類 | 内容 |
|------------|------|
| **個別契約** | 使いたい機能だけ月額で契約（例: 確率・抽選、自動返信、対人、表示位置） |
| **全機能パック** | 4 機能すべてをまとめて月額契約 |

**手順**

1. ① の登録が完了していることを確認（画面上部にメール・ユーザー名が表示される）  
2. 使いたい機能の **「契約する」** ボタン、または **「全機能パック」** のボタンを押す  
3. 表示される **Stripe の決済画面** でカード情報を入力して支払い  
4. オーバーレイ画面に戻ったら、該当機能が **「利用中」** になる  
5. 反映されない場合は **「状態を更新」** を押す  

**個別契約から全機能パックへ切り替える**

- すでに **個別で契約している** ときは、**「全機能パックに切り替え」** ボタンが表示されます  
- 切り替え後、Stripe 上の **個別契約は自動で解約** され、全機能パックに統一されます（アプリ側で二重課金にならないようになっています）  

**契約の確認・解約・カード変更**

- 課金パネルの **「契約の管理」** → Stripe の顧客ポータルが開きます  
- 解約やカード変更はそこで行います  

**料金の目安**（アプリ内の表示が正です）

- 個別: 各機能 **月 100 円**（表示と異なる場合は課金タブの表記を優先）  
- 全機能パック: 上記 4 機能分の合計（**月 400 円** 相当）  

---

#### ② 招待コードで使う場合（サブスク不要）

管理者から **招待コード（長い英数字の token）** をもらった場合:

1. ① と同じ手順で **アカウント登録**（メールは、管理者が登録したアドレスと **同じ** にする）  
2. **②-B 招待コード** に token を貼って **「適用」**  
3. **無期限** で PRO が使えます（月額契約は不要）  

招待コードは **メールアドレスではありません**。管理者から渡された文字列をそのまま貼ってください。

#### よくあるトラブル

| 症状 | 確認すること |
|------|----------------|
| ログインリンクが送れない | **「メール送信の上限」** と出たら 30〜60 分待つ（Supabase 無料枠は時間あたりの送信数に制限あり）。連打しない。開発者は Dashboard → Authentication → Rate Limits / Email を確認 |
| ログインリンクが送れない | **未登録のメール** で「ログイン」した場合 → **「新規登録」** タブを使う（invite 用メールだけ DB にある状態では Auth 未登録のことがあります） |
| ログインリンクが開けない | exe が起動中か、`http://localhost:4173/overlay` が開けるか |
| ユーザー名を保存できない（display_name / schema cache） | 管理者が Supabase に **未適用の SQL** がある。下記「DB マイグレーション」を参照 |
| 契約したのに PRO がロックのまま | **「状態を更新」** を押す／ブラウザを再読み込み |
| 招待コードが使えない | ① でログインしたメールが、管理者登録のメールと一致しているか |
| 全機能パックのボタンが出ない | すでに全機能契約済み、または ① のユーザー名が未保存 |

課金基盤（Supabase / Stripe）の管理者向け設定は [`docs/BILLING.md`](BILLING.md) を参照してください（一般ユーザーが設定する必要はありません）。

#### DB マイグレーション（管理者・`display_name` エラー時）

ユーザー名保存で `Could not find the 'display_name' column` と出る場合、Supabase に次の SQL が **まだ実行されていません**。

1. [Supabase Dashboard](https://supabase.com/dashboard) → 対象プロジェクト → **SQL** → **New query**
2. 次を貼り付けて **Run**:

```sql
alter table public.profiles
  add column if not exists display_name text;

comment on column public.profiles.display_name is 'ユーザー登録時に入力。課金・招待コード適用の前提';
```

3. 数秒待ってから、アプリで **「ユーザー名を保存」** を再試行

（ファイル: `supabase/migrations/20260330000002_profiles_display_name.sql`）
