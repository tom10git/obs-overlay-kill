================================================================================
  OBS Overlay Kill — 利用者向け（配布 exe）
================================================================================

■ 配布物

  OBS-Overlay-Kill.exe のみです（アプリ本体は exe 内に同梱され、
  横に dist フォルダやソースは付きません）。

■ 必要な環境

  • Windows
  • Node.js は不要

■ 使い方

  1. zip を展開し、OBS-Overlay-Kill.exe をダブルクリック
  2. 黒いウィンドウに URL が表示されたら準備完了
  3. OBS のブラウザソース URL（既定ポート 4173）:

       http://localhost:4173/overlay

  4. 終了はウィンドウで Ctrl+C

■ 設定・素材（利用者が触ってよい範囲）

  次のフォルダに保存されます（初回起動時、exe 内の設定・素材がここへコピーされます）。

    %LOCALAPPDATA%\OBS-Overlay-Kill\data\

    効果音 … data\src\sounds\
    画像・WebM … data\src\images\
    設定   … data\config\overlay-config.json
    カスタム技名 … data\config\customTechniqueNames.ts（任意）

  設定画面で「保存」やファイル参照を使うと、自動でこのフォルダに書き込まれます。
  起動時にコンソールへ実際のパスが表示されます。

  ■ 設定だけ初期化したいとき

    data\config\overlay-config.json を削除して exe を再起動すると、
    初期設定に戻ります（効果音・画像はそのまま残ります）。

  音や画像が鳴らない・表示されないときは、上記フォルダを一度削除して exe を
  再起動すると、同梱内容から再コピーされます（設定の再調整が必要な場合あり）。

■ ポート変更

    set OVERLAY_PORT=8080
    OBS-Overlay-Kill.exe

■ トラブル

  • ポート使用中 … OVERLAY_PORT で別ポートを指定
  • ウイルス対策が exe をブロック … 例外登録（pkg 製 exe は誤検知されやすい）
  • exe の横に dist フォルダを置いても読みません（古い配布形式は不要）
  • ブラウザで開けない … http://localhost:4173/overlay を直接開く／別ブラウザで試す
  • 画面が真っ黒・素材が読めない … いったん data フォルダを削除→再起動（同梱から再コピー）
  • 開発者が再ビルドできない … 実行中の OBS-Overlay-Kill.exe を閉じる（ビルド時は自動終了を試みます）
