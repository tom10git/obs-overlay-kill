================================================================================
  OBS Overlay Kill — ローカル配布パッケージ（release フォルダ）
================================================================================

■ このフォルダに含まれるもの

  dist/          … Web アプリのビルド結果（ビルド前に設定した JSON を含む）
  start-localhost.bat … Windows 用。ローカル HTTP でアプリを表示する
  README-配布.txt … この説明

■ 必要な環境

  • Windows 想定（.bat 用）
  • Node.js がインストールされていること（LTS 推奨）
      https://nodejs.org/
    ※ 初回起動時、npx が一時的に「serve」パッケージを取得します（ネット接続が必要）。
  ・画像や効果音は予め用意が必要
  　※distフォルダ内にご自身で各データに合わせたフォルダを用意していただき、その中に格納して設定で読み込むようにしてください。
  　　画像フォルダの例：dist/src/images/　ここにデータを格納
  　　効果音フォルダの例：dist/src/sounds/　ここにデータを格納

■ 使い方

  1. start-localhost.bat をダブルクリックする
  2. 黒いウィンドウに「配信開始」のような表示が出たら準備完了
  3. OBS を開き、ソースに「ブラウザ」を追加
  4. URL に次を入力（ポートは既定 4173。変えた場合は画面の表示に合わせる）

       http://localhost:4173/overlay

  5. 使用中は黒いウィンドウは閉じずにウィンドウを隠す
  6. 終了するときは、黒いウィンドウをアクティブにして Ctrl+C を押す

■ ポートを変えたい場合

  コマンドプロンプトで release フォルダに移動してから:

    set OVERLAY_PORT=8080
    start-localhost.bat

■ 開発者向け：このフォルダを作り直す

  リポジトリのルートで:

    npm install
    npm run package:release

  これでビルドが走り、release/dist が最新の内容に置き換わります。

■ トラブル

  • 「dist がありません」… 次が多いです:
    - 圧縮ファイル（zip 等）を展開せずに、中から .bat を直接実行している（いったん「すべて展開」してください）
    - 配布物に dist/ が含まれていない（作成者が package:release を実行していない）
    - release フォルダだけを移動して dist/ が置き去りになっている
    対処: release フォルダ直下に dist と .bat があるか確認。
    ※ もし「リポジトリ内の release」から実行している場合は、start-localhost.bat が
      `npm run package:release` を自動実行して dist を生成できることがあります。

  • ポートが既に使われている… 別アプリが 4173 を使用中。OVERLAY_PORT で変更。

  • npx でエラー… ネットワークまたは Node のインストールを確認。
